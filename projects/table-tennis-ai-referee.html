Table Tennis AI Referee

Project overview

Table Tennis AI Referee is an iOS-first, real-time single-camera officiating and scoring system that turns a side-view smartphone into an on-device referee. The app runs a full perception → tracking → physics inference → rules pipeline at interactive frame rates, detecting the ball, estimating a calibrated table coordinate frame from four user-tapped corners, and then inferring key events such as bounces, edge balls, and net touches. The design is explicitly physics-aware rather than just detection: ball motion is tracked continuously, smoothed, and interpreted as dynamics over time so the system can reason about trajectories, bounce candidates, and legality constraints in a stable state machine.

The computer vision stack is organized as a low-latency streaming pipeline. Frames come from a mobile camera feed (with a simulator/demo fallback), and a lightweight detection stage produces per-frame ball observations. Those observations feed a Kalman filter that estimates ball state (position and velocity) in image coordinates and provides per-frame residuals and covariance that become first-class signals for reliability. Table plane calibration is done via a homography computed from four corner correspondences, mapping image coordinates into normalized table coordinates. This enables consistent reasoning about which side (near vs far), in-bounds vs out-of-bounds, and boundary proximity independent of resolution and camera placement. Event inference operates over a short history window of tracked states; bounces are detected via velocity sign flips and geometric consistency with the calibrated table region, net touches are inferred via motion crossing near the net line with characteristic speed drops, and edge-ball likelihood is computed as a function of bounce location relative to table boundaries with an uncertainty-aware margin derived from calibration and tracking uncertainty.

A probabilistic call layer provides uncertainty estimates for each inferred event by combining detector confidence, Kalman innovation/residuals, and calibration uncertainty into interpretable probabilities (for example, "edge ball: 82%"). On top of event inference, a rules engine implements a simplified version of official scoring logic as a deterministic state machine with serve phases, deuce behavior, server alternation, let handling on net-touched legal serves, and conservative point attribution when some signals are ambiguous (because only a single camera is used and paddle contact is optional). The system maintains match state continuously (score, server, rally phase, and last outcome) and exposes structured reasoning strings so that every call can be explained in plain language.

The app experience is built around a guided workflow and transparent explainability. A setup wizard walks users through calibration with an alignment overlay and corner tapping, then transitions into live match mode with a scoreboard, serve indicator, and an AI referee overlay that renders the table outline, net line, ball track, inferred bounce/edge points, and per-event confidence with short textual reasons. A review/replay flow lets users inspect match outcomes, export structured match summaries, and scrub through a full match replay simulation that re-renders the table and trajectory timeline with event markers and score keyframes for post-match analysis or dispute resolution.

For evaluation and iteration, the project includes a labeling workflow and metrics harness designed for systematic improvement. Video clips can be annotated with ball positions and event frames (bounces and edges), exported as JSON, and fed into an evaluation pipeline that reports bounce detection accuracy, edge precision/recall, and latency/FPS-style performance metrics, including breakdowns by capture conditions such as lighting. A failure-case logger triggers automatically on low-confidence or ambiguous calls, saving metadata windows around uncertain events so problematic scenarios can be reviewed, replayed, and prioritized for model and pipeline improvements.

Overall, the project is structured as a production-oriented monorepo with a shared TypeScript core library for math, tracking, inference, and rules, and a mobile client that focuses on real-time camera integration, overlays, and UX. The architecture cleanly separates UI concerns from the inference engine, enabling rapid iteration of models and detectors while keeping the physics and rules logic stable and testable. The result is a robust, explainable, real-time officiating system that demonstrates end-to-end applied ML for mobile: calibration geometry, Bayesian tracking, uncertainty-aware event classification, and a rule-driven state machine, all designed to run on-device with interactive feedback.
