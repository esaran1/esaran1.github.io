Limit Order Book Trading Simulator

System overview and technical detail

The project implements a fully event-driven market microstructure simulation platform that reproduces the mechanics of modern electronic exchanges. At its core is a limit order book (LOB) with price-time priority matching, designed to serve as a controllable, reproducible environment for training and evaluating trading agents under realistic execution constraints.

The system is architected as a modular, distributed platform comprising a deterministic matching engine and market simulator, a reinforcement learning environment conforming to the Gymnasium API, a scalable training and evaluation pipeline, and a visual analytics dashboard for experiment inspection and replay. All components are containerized and orchestrated using Kubernetes, enabling reproducible experimentation and horizontal scaling of simulation and training workloads.

Limit order book simulator and matching engine

The simulator operates on a discrete event stream consisting of limit orders, market orders, order cancellations, and trade executions. Each event advances the system state deterministically, enabling exact replay and post-hoc analysis.

The LOB maintains bid and ask books indexed by price, FIFO queues at each price level to enforce price-time priority, and an order ID map to support efficient cancellation. Efficient data structures (such as sorted maps with per-price deques) ensure logarithmic updates while preserving deterministic behavior.

The matching engine supports partial and full fills, crossing orders, realistic spread dynamics, configurable latency between agent actions and exchange acknowledgment, and slippage due to queue position. Trade prints are recorded explicitly, preserving a full execution history for evaluation and replay.

The simulator can operate under multiple market regimes: stable, high-volatility, mean-reverting, and regime-switching environments. These regimes modify order arrival rates, price dynamics, and liquidity depth, allowing stress testing of agent robustness.

Reinforcement learning environment

At each timestep, the agent observes a compact yet information-dense state representation including top-k bid and ask levels, current spread and mid-price returns, agent inventory and unrealized PnL, and recent order book imbalance statistics. All observations are strictly causal to prevent future information leakage.

Agents act by placing limit orders with configurable price offsets and sizes, submitting market orders, or canceling outstanding orders. Rate limits and notional exposure caps are enforced to maintain realism.

The reward is formulated as risk-adjusted profit, combining realized PnL, inventory holding penalties, and adverse selection penalties during volatile regimes. This discourages trivial high-risk strategies and encourages stable execution behavior.

Baseline strategies (random, TWAP, naive market making) are implemented for benchmarking. The environment enforces no-cheat constraints, ensuring agents cannot observe future events or hidden simulator state.

Training, evaluation, and reproducibility

Training uses policy-gradient RL (PPO) with configurable episode lengths, seeded randomness for reproducibility, and distributed rollout workers. Experiments are orchestrated via a job queue and executed in isolated containers.

Evaluation focuses on distributional performance, not point estimates. Metrics include PnL distributions, maximum drawdown, inventory variance, execution quality (slippage vs mid-price), and robustness across regime shifts. Bootstrap confidence intervals are computed to assess statistical stability.

Each experiment logs simulator version and git commit hash, configuration files and random seeds, trained model artifacts, and evaluation outputs. Artifacts are stored in object storage, while metadata is tracked in a relational database, enabling exact experiment reproduction.

Replay and visualization dashboard

A web-based dashboard built with Next.js, Tailwind CSS, and shadcn-style UI provides experiment summaries and metric comparisons, time-series visualizations of prices, spreads, inventory, and PnL, and a replay interface that reconstructs order book states and agent actions over time. This allows qualitative inspection of agent behavior, failure modes, and execution dynamics beyond aggregate metrics.

Infrastructure and deployment

All services (simulator, training workers, API, dashboard) are containerized with Docker and deployed via Kubernetes using Kustomize overlays for development and production.

Horizontal Pod Autoscalers manage training and evaluation workloads. Resource limits and safety checks prevent runaway simulations. Health probes and network policies enforce service isolation.

The system exposes structured logs, Prometheus metrics, and OpenTelemetry traces to support debugging, performance analysis, and system monitoring.

Key contributions and novelty

This project goes beyond typical ML trading bots by modeling execution-level market mechanics rather than only price prediction, emphasizing offline policy evaluation and reproducibility, integrating simulation, RL, systems engineering, and visualization, and explicitly analyzing failure modes under distribution shift. The result is a research-grade experimental platform rather than a single trained model.

The Limit Order Book Trading Simulator demonstrates the ability to design realistic simulators, build RL environments with strict causality and evaluation discipline, engineer scalable, reproducible ML systems, and communicate complex agent behavior through interpretable tooling. This aligns closely with the technical expectations of quant trading firms, advanced ML teams, and top research institutions.
