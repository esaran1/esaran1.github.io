Speech ML Product

AccentCoach

Project overview

AccentCoach is a production-quality MVP I built to turn speech ML into something genuinely usable: a privacy-friendly web app that analyzes spoken English and provides supportive, word-by-word clarity coaching for accented speakers. Instead of generic advice, it gives actionable feedback at the exact moment a pronunciation issue likely occurred, and it pairs that feedback with audio so users can immediately compare what they said against a clear example. The product tone is intentionally encouraging, with messaging like "Accents aren't wrong. This is optional clarity coaching," and the UI is designed to feel modern and approachable rather than clinical.

Under the hood, AccentCoach is an end-to-end full-stack ML system that runs locally without paid APIs. On the backend (FastAPI + Uvicorn, Python 3.11), the app accepts either an uploaded audio file or a browser-recorded take. The audio is normalized using ffmpeg into a consistent mono 16 kHz WAV format to improve reliability across devices and file types. A background job system persists state to disk as JSON ("processing / done / error"), which keeps the experience responsive and resilient while making the pipeline reproducible and easy to debug. All artifacts including uploads, job metadata, word clips, and example audio are stored locally under a structured data directory.

The speech pipeline goes beyond basic transcription. The system transcribes speech and then aligns word timings to audio more precisely (using WhisperX alignment) so each word has reliable start and end timestamps. From those timestamps, AccentCoach slices the user's audio into per-word clips and attaches them to the transcript for interactive playback. For scoring, I implemented a phoneme-aware approach. The app derives an expected pronunciation (phoneme sequence) for each word and compares it against an observed phoneme estimate from the user's clipped audio. That comparison is combined with alignment confidence and accent-sensitive confusion patterns (for example TH substitutions, R/L confusions, V/W swaps, and stress or vowel length cues) to produce a 0-100 clarity score, an issue label, and a short coaching tip per word. The result is not just ASR confidence, but a more interpretable signal that maps to concrete, practiceable feedback. For flagged words, the system also generates an example pronunciation clip, with safe fallbacks if TTS is not available, so users can hear a clear reference immediately.

On the frontend (Next.js 14 App Router + TypeScript), I built a polished, accessible experience using Tailwind CSS and shadcn/ui. The results dashboard summarizes overall performance, highlights top recurring issues, and recommends a short practice list so users know what to work on next. The transcript renders as clickable word pills color-coded by score, and selecting a word opens a detail panel where users can play "You said" and "Example" audio with waveform visualization (WaveSurfer.js) for fast scrubbing and comparison. Subtle motion and thoughtful empty, loading, and error states keep the interaction smooth while the backend processes jobs asynchronously. Overall, AccentCoach demonstrates my ability to combine ML engineering, backend reliability, and product design into a cohesive system that feels like a real app, not a demo.
