Cricket Coach

Project overview

Cricket Coach is a local-first computer vision coaching system that analyzes short cricket clips and returns a replay with a pose overlay, key event markers, coaching metrics, and a structured JSON report. It is designed as a modular Python backend that can be ported to mobile later, with a polished web frontend for demonstration and review.

The backend uses MediaPipe Pose to extract 33 body landmarks per frame, then applies interpolation for short visibility gaps and EMA smoothing to stabilize motion. It supports two explicit analysis modes based on camera angle, side or rear-side and front-on, and keeps the logic and thresholds view-specific. A lightweight API accepts a video upload and returns a run id plus URLs to generated artifacts. Outputs include an overlay MP4 with a skeleton and event labels, a keypoints JSON containing raw and smoothed landmarks, and a report JSON containing events, metrics, tips, warnings, and quality statistics.

For analysis, the system implements heuristic event detection and coaching metrics per view. Side view focuses on front-foot plant detection, stride length, knee angle, and head stability, with additional bowling proxies when bowling mode is selected. Front view focuses on shoulder alignment, lateral head movement, symmetry cues, and experimental bowling elbow extension tracking. To make the metrics useful to coaches, the API layer converts raw numbers into coach-friendly outputs by normalizing key motion measures to percent of frame, attaching short explanations of why each metric matters, adding practical coaching cues, and providing a confidence score based on landmark visibility. Low-confidence metrics automatically generate warnings to prevent over-interpreting noisy signals.

To improve shot direction feedback beyond pose-only estimation, the system includes an optional ball tracking pipeline. It uses a two-stage approach with frame differencing and blob filtering to propose candidate ball positions, then refines the trajectory with optical flow to improve continuity when the ball is small or partially blurred. When tracking coverage is sufficient, the shot estimate is derived from ball motion rather than pose cues and the UI surfaces both the estimated fielder zone and the tracking coverage so the user can judge reliability.

The frontend is built with Next.js, Tailwind, and shadcn-style components. It provides an upload workflow with explicit view selection, mode selection, and optional batting context inputs like handedness and camera perspective. Results are presented with an overlay video player, metric cards that include status badges, recommendations, interpretation, coaching cues, and signal-quality indicators, plus a tips panel, warnings, and downloadable JSON links. Requests are proxied through the frontend to avoid cross-origin issues in local development.

Overall, this project demonstrates end-to-end applied ML engineering, from robust video IO and pose estimation through signal processing, heuristic event detection, confidence-aware metric design, and a clean UI that delivers actionable coaching feedback. It also shows engineering discipline through modular architecture, structured outputs, reproducible setup, and unit tests for core logic.
