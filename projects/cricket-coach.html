Cricket Coach

Project overview

Cricket Coach is a video analysis web app that helps cricket players improve technique using pose-based biomechanics. You upload (or record on mobile) a short clip, and it extracts body landmarks frame-by-frame, computes coaching metrics and movement quality indicators, and then gives you a simple “replay + numbers + tips” interface. The output is not just pose detection: you get overlays, structured reports, and phase-aware feedback.

Core user flow: record or upload a clip on the Home page, choose a camera view (side or front) and a mode (batting or bowling), then analyze. The backend runs pose estimation and analysis, returning overlay videos with guidance, a structured report (metrics, tips, warnings), and optional highlight/focus reels.

Pose extraction: the pipeline uses MediaPipe Pose to detect landmarks per frame. Short visibility gaps are handled with interpolation, and motion is stabilized with EMA smoothing. Analysis is view-specific: side vs front use different thresholds and baselines so the same pose landmark stream maps to coaching metrics that make sense for the camera.

Metric computation: landmarks get converted into angles, distances, and quality signals (alignment, movement stability, and phase-aware context). Metrics are benchmarked against view/mode baselines and labeled as good / ok / needs work. Coaching output is then generated as explanations and warnings so users don’t over-interpret noisy signals.

Kinematic sequence: it builds time-series segment speeds (pelvis → torso → shoulder → wrist), finds peak timing order, and flags “power leak” when the sequence looks out of order. That’s the part that turns raw motion into a coaching-ready story about what the body is doing through the action.

Bowling injury risk: the app includes bowling-side specific red-flag logic. It detects a mixed action pattern at the front-foot plant and release by comparing hip-line vs shoulder-line orientation, then surfaces it with a clear coaching explanation.

Pro comparison: for visual benchmarking, the app aligns your sequence with a selected pro using Dynamic Time Warping (DTW). It then renders a “ghost” overlay on top of your video so you can compare motion in the same time window.

Auto reels + voice coach: it can generate short highlight/focus clips from key event windows, and includes a “play coach summary” feature. When configured, the summary can be LLM-based (with a fallback), and it uses text-to-speech for quick guidance.

i18n: the UI supports multiple languages (including Hindi) through a locale provider and translation JSON files, with a responsive header language toggle.

Tech stack: backend is Python FastAPI with a MediaPipe + OpenCV + NumPy pipeline. Report generation, reel creation, and DTW/pro alignment run on the backend, and run metadata is persisted using SQLite. Frontend is Next.js/React with TypeScript and Tailwind for styling. Charts use Recharts, and the UI includes export/download controls and result tabs. On mobile, users can record inside the app using the browser MediaRecorder and then reuse the same upload/analyze pipeline.

What makes it distinctive: it’s not just “pose detection”—it turns pose landmarks into coaching-ready metrics, phase-aware feedback, and sequence-based power checks. You get both numbers + explanations and visual comparison (pro ghost overlay), with mobile-first usability and Hindi support for accessibility.
